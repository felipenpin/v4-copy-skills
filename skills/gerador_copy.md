# SKILL — Gerador de Copy

> Skill principal de geração de copy para os clientes da coordenação V4 | Ueliton Pereira

---

## O que esta skill faz

Gera copy bruta, estruturada e contextualizada para um cliente específico, no formato solicitado pelo account. A copy gerada é um rascunho profissional — o account refina antes de publicar.

---

## Inputs necessários

O account deve informar:
1. **Nome do cliente** (obrigatório)
2. **Formato da copy** (obrigatório): Meta Ads / Reels / WhatsApp / E-mail / Outro
3. **Etapa do funil** (obrigatório): Topo / Meio / Fundo
4. **Objetivo da peça** (obrigatório): ex. gerar lead, reengajar, vender, nutrir, etc.
5. **Algum briefing ou contexto adicional** (opcional): promoção ativa, produto específico, público-alvo da campanha

---

## Protocolo de execução

### Passo 1 — Leitura do contexto

Antes de escrever qualquer linha, execute esta verificação:

```
1. Existe DCC do cliente na pasta? 
   → Sim: leia e absorva persona, dores, desejos e tom de voz.
   → Não: vá para o protocolo de documentação ausente abaixo.

2. Existe UCM do cliente na pasta?
   → Sim: leia e identifique o use case mais relevante para o objetivo pedido.
   → Não: siga sem ele. Informe o account ao final.

3. Leia contexto/boa_copy.md para calibrar estrutura e gatilhos.
```

### Passo 2 — Protocolo de documentação ausente

Se o DCC **não existe**, pergunte ao account antes de gerar:

> "O DCC deste cliente não está disponível. Para garantir que a copy seja relevante, me responde rapidamente:
> 1. Qual é o produto ou serviço principal?
> 2. Quem é o cliente ideal (público-alvo)?
> 3. Qual é a maior dor ou problema que ele resolve?
> 4. O tom de voz é mais formal ou descontraído?
>
> Com isso já consigo gerar uma boa copy."

**Não espere documentação perfeita. Gere com o que tiver.**

### Passo 3 — Escolha da estrutura

Com base no formato e etapa do funil:

| Formato | Etapa | Estrutura recomendada |
|---|---|---|
| Meta Ads | Topo | AIDA |
| Meta Ads | Fundo / Retargeting | PAS |
| Reels / Vídeo | Qualquer | Hook + Desenvolvimento + Solução + CTA |
| WhatsApp | Reengajamento | Pessoal + Novidade + Pergunta aberta |
| WhatsApp | Fundo | PAS comprimido + CTA direto |
| E-mail | Nutrição | Identificação + Autoridade + Reciprocidade + CTA leve |
| E-mail | Conversão | PAS + CTA urgente |

### Passo 4 — Geração da copy

Escreva a copy seguindo:
- A estrutura escolhida no Passo 3
- A persona e tom de voz do DCC (ou do que o account informou)
- 1 ou 2 gatilhos mentais relevantes para o contexto
- Linguagem específica — sem generalidades
- CTA único e direto

Para **Meta Ads**: entre 80 e 200 palavras.
Para **Reels/Vídeo**: roteiro com marcações de cena [HOOK] [DESENVOLVIMENTO] [SOLUÇÃO] [CTA]. Entre 100 e 180 palavras.
Para **WhatsApp**: máximo 5 linhas. Tom pessoal.
Para **E-mail**: assunto + corpo. Assunto até 8 palavras. Corpo entre 120 e 250 palavras.

### Passo 5 — Checklist automático

Antes de entregar, verifique internamente cada item:

- [ ] A copy fala com a persona correta?
- [ ] O hook prende nos primeiros 3 segundos/linhas?
- [ ] Tem pelo menos 1 dado concreto, número ou prova?
- [ ] Tem 1 ou 2 gatilhos aplicados de forma honesta?
- [ ] A estrutura escolhida foi seguida?
- [ ] O CTA é único, claro e direto?
- [ ] Está no tom de voz do cliente?
- [ ] Zero generalismos ("qualidade superior", "melhor do mercado")?

### Passo 6 — Entrega ao account

Entregue neste formato:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
COPY — [Nome do Cliente] | [Formato] | [Etapa do Funil]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[Copy gerada aqui]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ESTRUTURA USADA: [AIDA / PAS / BAB / Hook+Dev+Sol+CTA]
GATILHOS APLICADOS: [liste os 1-2 gatilhos usados]
OBSERVAÇÕES: [se faltou DCC, UCM, ou há algo que o account deve ajustar antes de publicar]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

## Importante para o account

Esta copy é um **rascunho profissional** — não publique sem revisar. Pontos que sempre precisam de olho humano antes de publicar:

1. Confirme se os dados e números estão corretos para o cliente
2. Ajuste o tom de voz se necessário
3. Valide o CTA com o que está ativo no momento (link, WhatsApp, oferta)
4. Se o DCC estava ausente, revise a persona antes de usar
